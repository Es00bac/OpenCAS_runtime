Features

Subsystems that are present in the current code.

This page avoids aspirational claims and external comparisons. It describes the capabilities that exist now, including the newer voice, phone, schedule, platform, and telemetry surfaces.

Persistent memory

SQLite-backed episodic and distilled memory, edge graph storage, semantic retrieval, and artifact-backed autobiographical recall.

Retrieval fusion

Semantic, keyword, recency, salience, graph, emotional resonance, temporal echo, and reliability signals, plus somatic and relational adjustments.

Embeddings

Provider-backed embeddings through open_llm_auth, with google/embeddinggemma-300m as the current default native 768-dimensional local lane.

Somatic state

Arousal, fatigue, tension, valence, focus, energy, certainty, and a derived somatic tag that modulate pacing and retrieval.

Relational state

Musubi tracking and user/self-model continuity surfaces that influence interpretation and tone.

Executive state

Current intention, active goals, work queue, commitments, and persistent plans.

Creative ladder

Promotion from spark to note, artifact, micro-task, project seed, project, and durable work.

Daydreaming

Idle-time reflection, keeper selection, conflict tracking, and promotion of daydream-origin work.

Self-approval

Risk-tiered approval, evidence-sensitive escalation, and durable approval history.

Execution and salvage

Lane-limited BAA work, receipts, retry governance, and salvage-aware recovery that preserves blocked-vs-resumable state.

Scheduling

Durable task and reminder schedules, calendar views, and run history without pretending every event is a chat turn.

Platform and trust

Capability inventory, extension install/update flows, bundle inspection, and plugin trust policy control.

Tool-use memory

Compact records of which tools worked for similar tasks, paired with semantic tool routing so the agent can consult a relevant toolbox on demand.

Project return

Unfinished work can preserve its artifact, blocked state, attempt history, and next intended step so it can be resumed deliberately later.

Desktop context

An opt-in plugin packages explicit desktop context for grounded assistance without making hidden observation a default behavior.

Chat and voice

Session history, lane metadata, microphone transcription, and spoken reply synthesis in the dashboard chat surface.

Phone bridge

Twilio-backed owner and caller handling with workspace separation, screening, and recent-call inspection.

Telegram

Persisted Telegram configuration, pairing and DM policy control, typing indicators, edited replies, and media attachment context.
